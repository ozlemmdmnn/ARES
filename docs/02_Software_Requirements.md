# Software Requirements Specification (SRS)

## 1. Introduction

ARES is an AI-assisted disaster management application.

The first version (MVP) focuses on disaster reporting, risk analysis and emergency resource management.

---

## 2. Functional Requirements

### FR-01 User Login

Users can log into the system.

### FR-02 Disaster Reporting

Users can report disasters.

Information includes:

- Disaster Type
- Location
- Description
- Severity Level

### FR-03 Risk Analysis

The system calculates a risk score based on the entered information.

### FR-04 Emergency Resources

The system displays available emergency resources.

### FR-05 Incident History

Users can view previous incidents.

---

## 3. Non-Functional Requirements

- Fast response
- Easy to use
- Reliable
- Secure
- Scalable
